#### 1. Future — асинхронные операции

**Где используется:**

**a) Асинхронная инициализация игры** (`game_logic.dart`)
```dart
Future<void> initializeAsync(bool vsAI, String modeName) async {
  print('⏳ Инициализация игры...');
  // Future.delayed — имитация асинхронной загрузки конфигурации
  await Future.delayed(Duration(milliseconds: 300));
  // ... создание полей, запуск логирования
  await eventStream.startFileLogging('game_logs');
  print('✅ Игра инициализирована!');
}
```
Метод `initializeAsync()` использует `Future.delayed` для имитации загрузки ресурсов и `await` для ожидания создания файла лога.

**b) Асинхронное размещение кораблей с прогресс-баром** (`game_logic.dart`)
```dart
Future<bool> placeShipsRandomlyAsync(GameBoard board, int player) async {
  for (int i = 0; i < sortedShips.length; i++) {
    // Future.delayed — пауза между размещениями для визуализации
    await Future.delayed(Duration(milliseconds: 150));
    board.placeShipRandom(shipSize, random);
    // Прогресс-бар обновляется после каждого корабля
    stdout.write('\r  ⏳ Игрок $player: [...] $progress%');
  }
}
```
Каждый корабль размещается с задержкой через `Future.delayed`, что позволяет отображать анимированный прогресс-бар.

**c) Загрузка статистики предыдущей игры** (`statistics.dart`)
```dart
static Future<String?> loadLastStatistics(String directoryPath) async {
  final directory = Directory(directoryPath);
  if (!await directory.exists()) return null;
  // Stream из Directory.list() для перечисления файлов
  await for (final entity in directory.list()) {
    if (entity is File && entity.path.endsWith('.txt')) {
      files.add(entity);
    }
  }
  // Асинхронное чтение самого свежего файла
  final content = await files.first.readAsString();
  return content;
}
```
Метод `loadLastStatistics()` асинхронно ищет и читает файл с результатами предыдущей игры.

**d) Неблокирующая задержка ИИ** (`main.dart`)
```dart
// Вместо sleep() (блокирует поток) используем Future.delayed (не блокирует)
await Future.delayed(Duration(milliseconds: 500));
```
Замена `sleep()` на `Future.delayed()` — ключевое отличие: `sleep` блокирует весь поток, а `Future.delayed` позволяет event loop обрабатывать другие события.

#### 2. Stream — потоковая обработка событий

**Новый файл: `lib/game_event_stream.dart`**

**a) Система событий с StreamController**
```dart
class GameEventStream {
  // Broadcast StreamController — несколько подписчиков одновременно
  final StreamController<GameEvent> _controller =
      StreamController<GameEvent>.broadcast();

  // Список событий для финального отчёта
  final List<GameEvent> _eventLog = [];

  Stream<GameEvent> get stream => _controller.stream;

  void addEvent(GameEvent event) {
    _eventLog.add(event);
    if (!_controller.isClosed) {
      _controller.add(event);
    }
  }
}
```
`StreamController.broadcast()` создаёт поток, на который могут подписаться несколько слушателей одновременно. Каждое игровое действие (выстрел, уничтожение корабля, смена хода) генерирует событие.

**b) Типы событий (наследники GameEvent)**
| Класс | Описание |
|-------|----------|
| `ShotEvent` | Выстрел игрока (координаты, результат) |
| `ShipDestroyedEvent` | Уничтожение корабля (владелец, размер) |
| `TurnChangeEvent` | Смена хода (номер следующего игрока) |
| `GameOverEvent` | Завершение игры (победитель, всего выстрелов) |
| `ShipPlacedEvent` | Размещение корабля (игрок, размер) |
| `AnalysisEvent` | Результат анализа поля |

**c) Фильтрация потока по типу**
```dart
Stream<T> whereType<T extends GameEvent>() {
  return _controller.stream.where((event) => event is T).cast<T>();
}
Stream<ShotEvent> get shotEvents => whereType<ShotEvent>();
Stream<ShipDestroyedEvent> get destroyedEvents => whereType<ShipDestroyedEvent>();
```
`stream.where()` и `stream.map()` — трансформации потока для получения только нужных событий.

**d) Логирование в файл через StreamSubscription**
```dart
Future<void> startFileLogging(String directoryPath) async {
  _logSink = file.openWrite();
  // Подписка на Stream — каждое событие записывается в файл
  _fileLogSubscription = _controller.stream.listen(
    (event) { _logSink?.writeln(event.toString()); },
    onError: (error) { _logSink?.writeln('[ERROR] $error'); },
  );
}
```
`stream.listen()` создаёт подписку, которая автоматически записывает каждое событие в лог-файл.

**e) Подписка в main.dart**
```dart
// Подписка на уничтожение кораблей для уведомлений
game.eventStream.destroyedEvents.listen((event) {
  print('  🔔 [Stream] ${event.description}');
});
```
При каждом уничтожении корабля подписчик выводит уведомление в консоль.

#### 3. Isolate — параллельные вычисления

**Новый файл: `lib/isolate_ai.dart`**

**a) Вычисление хода ИИ в отдельном изоляте**
```dart
Future<AiComputeResult> computeAiMoveInIsolate(AiComputeData data) async {
  // Isolate.run() создаёт новый изолят, выполняет функцию и возвращает результат
  final result = await Isolate.run(() => _computeBestMove(data));
  return result;
}
```
`Isolate.run()` (Dart 2.19+) — самый простой способ запустить вычисления в отдельном потоке. Данные автоматически копируются между изолятами (нет общей памяти).

**b) Стратегический расчёт хода (top-level функция)**
```dart
// Top-level функция — обязательное требование для Isolate
AiComputeResult _computeBestMove(AiComputeData data) {
  // Стратегия 1: Продолжение поиска корабля рядом с попаданием
  // Стратегия 2: "Тепловая карта" вероятностей
  // Стратегия 3: Шахматный паттерн
  return AiComputeResult(
    row: chosen[0], col: chosen[1],
    confidence: confidence,  // Уверенность 0.0 - 1.0
    reasoning: reasoning,    // Объяснение выбора
  );
}
```
Функция `_computeBestMove` — top-level (не метод класса), что является обязательным требованием для передачи в `Isolate.run()`.

**c) Сериализация данных для Isolate**
```dart
class AiComputeData {
  final int boardSize;
  final List<List<int>> boardState; // 0=empty, 1=ship, 2=hit, 3=miss
  final List<List<int>> hitPositions;
  final List<List<int>> missPositions;
}
```
Isolate не может работать с произвольными объектами — только с примитивными типами. Поэтому `GameBoard` конвертируется в `List<List<int>>`.

**Новый файл: `lib/board_analyzer.dart`**

**d) Анализ поля в Isolate**
```dart
Future<BoardAnalysisResult> analyzeBoardInIsolate(BoardAnalysisData data) async {
  final result = await Isolate.run(() => _performAnalysis(data));
  return result;
}
```
Второй Isolate анализирует поле и возвращает:
- Процент покрытия
- Точность стрельбы
- Карту опасных зон
- Рекомендацию для игрока
- Оценку оставшихся кораблей
- Прогресс игры

Игрок может ввести команду "анализ" во время своего хода для получения подсказки.

### 🎯 КЛЮЧЕВЫЕ КОНЦЕПЦИИ

#### Future
- **Что это**: объект, представляющий значение, которое будет доступно в будущем
- **async/await**: синтаксический сахар для работы с Future
- **Future.delayed**: создаёт Future, завершающийся через заданное время
- **Применение в проекте**: инициализация, размещение кораблей, загрузка файлов, задержка ИИ

#### Stream
- **Что это**: последовательность асинхронных событий
- **StreamController**: создаёт и управляет потоком событий
- **broadcast**: позволяет нескольким подписчикам слушать один поток
- **Применение в проекте**: трансляция игровых событий, логирование, уведомления

#### Isolate
- **Что это**: независимый поток выполнения с собственной памятью
- **Isolate.run()**: простой API для запуска вычислений и получения результата
- **Ограничения**: передаются только примитивные/сериализуемые данные
- **Применение в проекте**: расчёт хода ИИ, анализ игрового поля
